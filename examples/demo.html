⚜ Lys

A structural slide engine for LLM-generated presentations

What is Lys?

  • Zero-dependency CSS + TypeScript library
  • Turns <article> elements into slide presentations
  • Visually unopinionated — bring your own theme
  • Behaviorally opinionated — navigation, a11y, presenter sync

The HTML Contract

<div data-lys>
  <article>
    <h1>First Slide</h1>
    <p>Hello, world.</p>
  </article>
  <article>
    <h2>Second Slide</h2>
    <p>That's it. That's the API.</p>
  </article>
</div>

Progressive Enhancement

No JS, no CSS

Articles flow as a readable document.


CSS only

Scroll-snap slides, navigable by scrolling.

CSS + JS

Full experience: keyboard navigation, state tracking, programmatic API, presenter mode.


Size budget

JS < 5 KB gzip. CSS < 2 KB gzip. Currently ~1.2 KB total.

Data Attributes

<article
  data-class="title-slide dark"
  data-background="linear-gradient(135deg, #0f0c29, #302b63)"
  data-notes="Speaker notes go here"
  data-transition="fade"
  data-timing="30"
>
  <h1>Your Content</h1>
</article>

Design Tokens

  • Two-tier system: --lys-* (public) / --_lys-* (internal)
  • Override at :root, container, or individual slide level
  • 9 tokens: aspect ratio, padding, transitions, font size, gap, dimensions, focus ring
  • prefers-reduced-motion sets transitions to 0ms automatically

LLM-First Design

  • Flat structure — slides are siblings, not nested
  • Standard HTML elements only — nothing proprietary
  • Single self-contained HTML file as output format
  • Auto-initializes — zero setup code needed
  • Graceful degradation — wrong attributes don't break slides
